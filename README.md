# 🔐 Backup Tool

Утилита для безопасного резервного копирования данных с гибридным шифрованием (Curve25519 + AES-256-GCM) и логированием операций.  
Разработана на языке Rust в рамках курсовой работы по системному программированию.

## 📦 Возможности

- Рекурсивный обход выбранного каталога (символические ссылки игнорируются)
- Упаковка всех файлов в сжатый **tar.gz** архив
- Гибридное шифрование:
  - Эфемерные ключи **X25519** (Elliptic Curve Diffie‑Hellman)
  - Аутентифицированное симметричное шифрование **AES-256-GCM**
- Безопасное хранение сеансового ключа – зашифрованный конверт (`*.key`)
- Подробное логирование с настраиваемым уровнем и автоматической ротацией файлов
- Консольный интерфейс — две подкоманды `backup` и `restore`
- Конфигурация в формате TOML

## 🛠 Сборка

Для сборки необходим Rust не ниже версии 1.70.

```bash
git clone <url-репозитория>
cd backup_tool
cargo build --release
```

В папке `target/release` появится исполняемый файл `backup_tool` (и вспомогательный `gen_keypair`).

## 🚀 Быстрый старт

### 1. Генерация ключевой пары получателя

```bash
cargo run --bin gen_keypair
# или ./target/release/gen_keypair
```

Появятся файлы `recipient_secret.key` и `recipient_public.key` (32 байта каждый).  
**Секретный ключ** следует хранить отдельно от резервных копий.

### 2. Подготовка тестовых данных и конфигурации

Создайте каталог с файлами:

```bash
mkdir -p testdata/docs
echo "Important document" > testdata/docs/secret.txt
echo "Another file" > testdata/notes.txt
```

Пример конфигурации `config.toml`:

```toml
source_dir = "testdata"
dest_dir = "backups"
recipient_public_key = "recipient_public.key"
log_file = "backup_tool.log"
max_log_size_mb = 5
```

Создайте папку для бэкапов:

```bash
mkdir backups
```

### 3. Создание резервной копии

```bash
./target/release/backup_tool backup --config config.toml
```

После выполнения в папке `backups` появятся два файла с временной меткой:  
- `backup_YYYYMMDD_HHMMSS.tar.gz.enc` — зашифрованный архив  
- `backup_YYYYMMDD_HHMMSS.key` — зашифрованный конверт с сеансовым ключом

Лог сохраняется в файл, указанный в `log_file`.

### 4. Восстановление данных

```bash
# Создайте каталог для восстановленных файлов
mkdir restored

# Подставьте актуальные имена файлов
./target/release/backup_tool restore \
    --config config.toml \
    --secret-key recipient_secret.key \
    --enc-key backups/backup_20260427_044048.key \
    --input backups/backup_20260427_044048.tar.gz.enc \
    --output restored
```

Проверьте целостность:

```bash
diff -rq testdata restored
```

(Ожидается пустой вывод или только предупреждение о пропущенных симлинках.)

## 🔐 Криптографические детали

1. При создании бэкапа генерируется эфемерная пара ключей X25519.
2. С помощью открытого ключа получателя вычисляется общий секрет (ECDH).
3. Из общего секрета через HKDF-SHA256 формируется ключ шифрования AES-256 и nonce.
4. Сжатый архив шифруется алгоритмом AES-256-GCM (с аутентификацией).
5. Сеансовый ключ дополнительно заворачивается (wrap) тем же производным ключом и сохраняется в конверте вместе с эфемерным публичным ключом.
6. При восстановлении процедура обратна: общий секрет восстанавливается из эфемерного публичного ключа и закрытого ключа получателя, затем расшифровывается сеансовый ключ, и им расшифровывается архив.

Используются проверенные крейты сообщества [RustCrypto](https://github.com/RustCrypto).

## 📋 Логирование

- Сообщения записываются в файл, заданный в `log_file`
- Формат: `[YYYY-MM-DDTHH:MM:SSZ LEVEL module] message`
- При превышении порога `max_log_size_mb` (по умолчанию 5 МБ) файл лога переименовывается с добавлением временной метки, создаётся новый
- Хранится не более 5 поколений старых логов
